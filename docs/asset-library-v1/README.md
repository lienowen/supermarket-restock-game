# 超市补货游戏：全局素材库设计 v1.0

本规范用于 `supermarket-restock-game` 的批量美术生产、素材验收、运行时注册和发布裁剪。

## 1. 核心原则

- 素材标准化：同类素材统一画布、比例、锚点、透视、光向和阴影。
- 能力组件化：素材描述由运行时目录统一解析，Scene 和关卡不直接写文件路径。
- 状态数据化：空、少、半满、较满、满、错放、打开、损坏、断电等均作为状态或变体。
- 表现与逻辑分离：替换美术资源不修改玩法规则。
- 源库与发布包分离：仓库可保留完整源素材库，正式构建只发布当前关卡实际引用的资源。

## 2. 与现有代码的对齐

现有项目继续使用：

- `src/game/assets/starterAssetCatalogue.ts`：运行时素材键、路径、尺寸、锚点、层级和状态的唯一入口。
- `public/assets/game/...`：运行时素材根目录。
- 内容配置只保存 `assetKey`，不得在关卡和 Scene 中直接写素材路径。
- `concept / prototype / production / deprecated`：素材生命周期。
- 固定第三人称透视、左上光向和统一深度组。

建议新增目录：

```text
public/assets/game/
├─ production-v2/
│  ├─ environments/
│  ├─ fixtures/
│  ├─ characters/workers/
│  ├─ characters/customers/

```

## 4. 生产尺寸

| 类型 | 建议源画布 | 默认锚点 | 说明 |
|---|---|---|---|
| 环境 | 2048×1536 或 1920×1080 | `[0,0]` | 不烘焙交互箭头和 UI |
| 货架/设备 | 1024×1024 | `[0.5,0.96]` | 底部落地，统一左上光源 |
| 角色/动作 | 768×768 | `[0.5,0.96]` | 所有动作脚底位置一致 |
| 商品 | 512×768 | `[0.5,0.96]` | 同品类比例统一，底部对齐 |
| 道具 | 768×768 | `[0.5,0.96]` | 空/满/打开状态共用主体轮廓 |
| UI | 逻辑尺寸的 2× | 组件决定 | 文本与图形分离，支持多语言 |
| 特效 | 256/512 方形 | `[0.5,0.5]` | 循环特效标注帧率 |
| 材质 | 512×512 | `[0.5,0.5]` | 必须无缝平铺 |
| 音频 | 44.1/48 kHz | — | OGG 为主，必要时提供 MP3 |

## 5. 命名规范

运行时 Key：

```text
category-subcategory-subject-variant-state-view
```

示例：

```text
product-beverage-cola-bottle-a-single
fixture-cooler-beverage-a-full
character-worker-a-restock-high
ui-hud-task-panel-default
```

文件路径：

```text
assets/game/production-v2/<category>/<subject>/<name>.<ext>
```

禁止：

- 用 `day01`、`level3` 命名可复用素材。
- 文件名使用空格、中文、括号和日期。
- UI 文字直接烘焙进按钮或面板。
- 同一个 Key 表达多个不同语义。

## 6. 状态模型

每个可交互对象拆成：

```text
主体 identity + 视觉变体 variant + 运行状态 state
```

示例：

```text
fixture: beverage-cooler-a
variant: blue-frame
state: empty | low | half | high | full | wrong-product
       | open | iced | broken | power-off
```

状态图应复用同一主体轮廓，不为每个关卡复制一套货架或冷柜。

## 7. 深度组

```text
far-environment → architecture → fixtures → fixture-contents
→ props → actors → world-effects → ui
```

新增素材必须声明深度组，Scene 不临时猜测层级。

## 8. 加载与发布

- `core-ui`：启动必需资源。
- `starter-market`：首关场景、员工、核心货架和少量商品。
- `starter-market-customers`：顾客按需延迟加载。
- `mission-*`：清洁、找货、收银等任务专属资源按任务加载。
- 概念图、源文件、未使用变体不得进入 `dist`。

## 9. 当前五类玩法的 P0 顺序

1. 补货：空闲、行走、推车、搬箱、开箱、低/中/高层补货、纸箱、推车、冷柜库存状态。
2. 促销补货：水、可乐、促销牌、倒计时、连击和金币特效。
3. 收银：收银台、扫描器、顾客排队/购物篮/付款及扫描音效。
4. 清洁：拖把、警示牌、污渍等级、泡沫和湿滑地面。
5. 找货：牛奶、苹果、麦片、任务清单、选中高亮和错误反馈。
