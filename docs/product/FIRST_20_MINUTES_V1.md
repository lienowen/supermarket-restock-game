# 前 20 分钟商业体验标准 v1

本文件定义首发版本的玩家体验目标。构建通过、没有报错和自动化测试通过是发布底线，不是产品完成标准。

## 1. 产品目标

玩家在前 20 分钟内应形成完整认知：

1. 我知道现在要做什么；
2. 操作立即产生清晰、舒服的反馈；
3. 完成工作会赚钱、涨星和推进店铺经营；
4. 每一关都引入一个新变化，而不是重复点击；
5. 我愿意继续下一关，看看店铺还能变成什么样。

## 2. 时间线

### 0—10 秒：第一眼

- 画面必须像一个完整超市，不像素材拼接页。
- HUD 只突出当前任务、进度、金币和星级。
- 玩家不阅读长说明也能看懂首个目标。
- 当前可交互目标有唯一且明确的视觉焦点。

### 10—60 秒：第一次成功

- 玩家必须移动角色，再完成取箱、装车、开箱和第一次补货。
- 每一步有动作、音画反馈和任务文案变化。
- 第一次补货应出现明显的成功反馈，不使用阻挡玩法的大弹窗。
- 错误操作应说明“为什么错”，不能只扣分或震屏。

### 1—5 分钟：形成爽点

- 连续正确补货形成节奏、连击或时间压力。
- 金币、星级和进度变化必须可见、可理解。
- 完成关卡后展示成绩、奖励和下一关的新内容。
- 继续按钮是结算界面的唯一主操作。

### 5—12 分钟：玩法变化

- 第二关不能只是更换商品图片，应改变目标节奏或任务条件。
- 第三关进入收银玩法，展示顾客、扫描和服务反馈。
- 玩家已理解：不同工作都在推动同一家店铺成长。

### 12—20 分钟：经营期待

- 清洁和找货关卡提供不同操作节奏。
- 关卡间保留金币、星级和声望，形成连续经营感。
- 至少出现一次清晰的升级、解锁或店铺改善预告。
- 第 20 分钟结束时，玩家应有明确的下一目标，而不是只看到重复关卡。

## 3. 画面与 UI 验收

- 环境、角色、货架、商品和 UI 使用统一透视、光向和色彩层级。
- HUD 不遮挡核心交互区域；触控区域不低于 44 逻辑像素。
- 文本与面板分离，禁止把任务文字烘焙进图片。
- 普通、可操作、按下、成功、错误和禁用状态必须可区分。
- 动画服务于理解和爽感，禁止长时间循环闪烁或无意义漂浮。
- 金币、星级、进度和任务变化需要过渡反馈，不能突然跳数。
- 结算界面必须呈现成绩、奖励、下一目标和唯一主按钮。

## 4. 玩法验收

每个关卡都按玩家视角回答：

- 三秒内能否判断当前目标？
- 玩家是否知道该点哪里、走到哪里？
- 正确操作是否立即得到正反馈？
- 错误操作是否能理解并立刻纠正？
- 这一关相对上一关新增了什么体验？
- 完成后为什么愿意进入下一关？

任何一项回答不清楚，关卡都不能只以“没有 Bug、编译通过”判定完成。

## 5. P0 实施顺序

1. 统一五关 HUD、任务信息和奖励反馈；
2. 打磨首关 0—60 秒引导与操作节奏；
3. 打磨结算到下一关的期待感；
4. 检查五关环境、角色和道具比例；
5. 补声音、震动、成功和错误反馈；
6. 最后才增加新素材、新关卡和商业化入口。

## 6. 发布门槛

上线前必须同时满足：

- `npm run release:check` 通过；
- 五关浏览器 UI 回归通过；
- 首关完整人工体验无迷路、误点或信息冲突；
- 常见手机和桌面横屏不遮挡核心玩法；
- 首包只加载当前关卡必需素材；
- 所有运行时素材在 Asset Catalogue 中登记且文件真实存在；
- 人工评审确认前 20 分钟不是重复点击，而是连续的经营体验。
